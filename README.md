## 📖 Summer Analytics {GeeksforGeeks X Salesforce}

Welcome to my repository for **Summer Analytics 2025**! This repository tracks my learning journey, code assignments, and hackathon submissions throughout the intensive 5-week bootcamp. The course covers everything from core data manipulation and exploratory data analysis (EDA) to advanced supervised machine learning and capstone project implementations.

---

## 📂 Weekly Breakdown

Below is a detailed outline of the assignments and projects completed week by week:

### 🔹 Week 2: Classification & Logistic Regression
Focus on binary classification methodologies and building end-to-end predictive pipelines.
- 📓 **[`Full_Logistic_Model.ipynb`](./Week%202/Full_Logistic_Model.ipynb)**: An end-to-end logistic regression model encompassing data cleaning, feature engineering, and model evaluation.
- 📊 **Datasets**: `hacktrain.csv` (Training data) and `hacktest.csv` (Testing data).
- 📤 **Output**: `submission.csv` containing the final classification predictions for the mini-hackathon.

### 🔹 Week 3: Data Analysis & Exploration
Emphasis on exploratory data analysis, plotting, and extracting insights from raw datasets.
- 📓 **[`SA2025_W1.ipynb`](./Week%203/SA2025_W1.ipynb)**: Core EDA notebook focusing on data manipulation and visualization techniques.
- 📊 **Dataset**: `Cars.csv` used for hands-on analysis.
- 📚 **Resources**: `Summer Analytics 2025-2.pdf` detailing the week's curriculum and objectives.

### 🔹 Week 4: Supervised Learning Hackathon
An intensive week dedicated to testing machine learning skills in a competitive format.
- 📓 **[`ok.ipynb`](./Week%204/ok.ipynb)**: The primary experiment and execution notebook containing model training iterations and hyperparameter tweaks.
- 📊 **Datasets**: `Train_Data.csv` (Historical features) and `Test_Data.csv` (Validation set).
- 📤 **Output**: `submission.csv` with final modeled outputs.

### 🔹 Week 5: Capstone Project (Dynamic Pricing Engine)
The final real-world capstone: Building a dynamic pricing engine simulation for urban parking lots utilizing both baseline and demand-based models.
- 📓 **[`main.ipynb`](./Week%205/main.ipynb)**: The central computation notebook calculating optimal prices based on various real-time parameters.
- 📄 **Documentation**: Sub-directory `README.md` providing architectural context on the pipeline.
- 📊 **Dataset**: `dataset.csv` containing massive urban parking log records.
- 📤 **Output**: `baseline_pricing_output.csv` featuring the simulated dynamically adjusted prices.
